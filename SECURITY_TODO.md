# MiniOC 安全升級待辦事項

> 本文件定義 MiniOC 專案的安全漏洞修復清單，供所有 AI Agent 執行任務時參考。

---

## 嚴重性分類

| 等級 | 說明 |
|------|------|
| 🔴 Critical | 立即修復，可能導致資料外洩或系統被入侵 |
| 🟠 High | 應儘速修復，存在安全風險 |
| 🟡 Medium | 建議修復，提升整體安全性 |
| 🟢 Low | 優化項目，非緊急 |

## 待辦清單

### 1. 🔴 Critical: 敏感資訊洩漏 - .env 檔案

**問題描述：**
- `.env` 檔案包含真實的 API Key 和 Bot Token
- 這些敏感資訊可能已被提交到 Git 歷史記錄中

**修復步驟：**
1. 立即輪換所有暴露的憑證：
   - `TELEGRAM_BOT_TOKEN`
   - `MINIOC_API_KEY`
2. 使用 Git BFG Repo-Cleaner 或 git filter-branch 清除 Git 歷史記錄中的敏感資料
3. 確保 `.gitignore` 已正確配置 `.env` 檔案
4. 建立新的 `.env` 檔案

**參考檔案：** `.env`, `.gitignore`

---

### 2. 🔴 Critical: 命令注入漏洞

**問題描述：**
`src/ai.js:79` 使用 `sh -c` 執行命令，使用者輸入的 prompt 未經過濾，可能被注入惡意指令。

```javascript
// 危險程式碼
const command = `yes | opencode ${args.map(a => `"${a.replace(/"/g, '\\"')}"`).join(' ')}`;
const child = spawn('sh', ['-c', command], { env: customEnv });
```

**修復步驟：**
1. 避免使用 `sh -c`，改用陣列形式傳遞參數
2. 對 prompt 進行輸入驗證和清理
3. 實作命令執行逾時機制
4. 限制允許的參數值

**參考檔案：** `src/ai.js:62-82`

---

### 3. 🟠 High: SQL 注入防護

**問題描述：**
`src/db.js` 中 `userId` 參數未進行類型驗證，雖然使用 prepared statements，但建議加強驗證。

**修復步驟：**
1. 對 `userId` 進行類型檢查（應為正整數）
2. 對 `content` 進行長度限制
3. 考慮對特殊字元進行過濾
4. 增設 SQL 查詢日誌監控

**參考檔案：** `src/db.js:45-72`

---

### 4. 🟠 High: 缺少速率限制 (Rate Limiting)

**問題描述：**
- 沒有對使用者訊息頻率進行限制
- 可能遭受洪水攻擊或 API 濫用

**修復步驟：**
1. 實作每使用者每分鐘訊息上限（如：每分鐘 10 則）
2. 實作每使用者每日 API 呼叫配額
3. 對超出限制的使用者返回友好提示
4. 記錄異常行為以便分析

**參考檔案：** `src/bot.js`

---

### 5. 🟡 Medium: 輸入驗證與過濾

**問題描述：**
- 使用者訊息沒有長度限制
- 沒有過濾可能導致 Markdown/HTML 注入的特殊字元

**修復步驟：**
1. 限制訊息最大長度（如 4000 字元）
2. 過濾危險的 Markdown/HTML 標籤
3. 對 `role` 欄位進行白名單驗證
4. 清理輸出回應中的控制字元（已有部分處理）

**參考檔案：** `src/bot.js:150-202`, `src/db.js`

---

### 6. 🟡 Medium: HTTPS 強制使用

**問題描述：**
`src/ai.js` 中的 API 呼叫使用 `https` 模組，但沒有明確驗證憑證。

**修復步驟：**
1. 驗證 API 回傳的 TLS 憑證
2. 考慮使用固定 CA 憑證
3. 記錄 TLS 連線資訊以便稽核

**參考檔案：** `src/ai.js:9-60`

---

### 7. 🟡 Medium: 錯誤處理資訊洩漏

**問題描述：**
- 錯誤訊息可能暴露內部系統資訊
- 堆疊追蹤可能暴露檔案路徑等敏感資訊

**修復步驟：**
1. 對外錯誤訊息應該是通用的
2. 詳細錯誤只記錄到日誌，不回傳給使用者
3. 實作統一的錯誤處理機制

**參考檔案：** `src/bot.js:184-201`, `src/ai.js`

---

### 8. 🟢 Low: 日誌安全

**問題描述：**
- 可能記錄敏感資料（如 API Key 部分內容）
- 沒有日誌級別控制

**修復步驟：**
1. 實作日誌級別設定（debug, info, warn, error）
2. 自動過濾日誌中的敏感資訊
3. 考慮將日誌輸出到安全的儲存系統

**參考檔案：** `src/logger.js`

---

### 9. 🟢 Low: 鎖定檔案安全

**問題描述：**
`src/index.js` 中的鎖定檔案可以被任意刪除或修改。

**修復步驟：**
1. 使用程序 ID 驗證鎖定是否有效
2. 實作檔案權限控制
3. 記錄鎖定相關的所有操作

**參考檔案：** `src/index.js:10-45`
